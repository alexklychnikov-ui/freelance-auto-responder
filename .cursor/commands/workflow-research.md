# Этап: research

Режим **только исследование**. Запрещено: писать код, предлагать исправления/дизайн решений, придумывать факты.

Соблюдай `.cursor/rules/research-phase-constraints.mdc` и выходной gate из `ai/workflow/stage-policy.json` для этапа `research`.

Действия:
- Собери факты из репозитория, явных входных данных пользователя и официальных доков (если даны ссылки).
- Если задача связана с пользовательскими данными/знаниями: выполни поиск по данным в LightRAG, используя доступные материалы и конфигурацию из `C:\Python\Projects\LigthRAG` как дополнительный источник фактов.
- Раздели: подтверждённые факты vs гипотезы (гипотезы помечай явно).
- Зафиксируй список неоднозначностей / вопросов к пользователю.

Выход: `fact_log`, `ambiguity_questions`. По закрытию этапа — checkpoint в `logs/agent-worklog.txt`.

Если пользователь описал новую задачу целиком — работай только в рамках research; не переходи к design до явного закрытия gate.
